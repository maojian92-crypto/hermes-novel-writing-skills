# Truth Files 长期一致性管理指南

> 主服务: 跨章节/跨卷设定一致性
> 内容层级: 防漂移基础设施

---

## 什么是 Truth Files

Truth Files 是小说项目的"设定圣经"，记录所有已确立的设定、角色状态、任务进度、谜团伏笔。任何新创作必须以此为锚，不得凭空创造。

---

## Truth Files 清单

| 文件 | 内容 | 更新时机 |
|------|------|----------|
| `01-核心设定与角色.md` | 核心世界观、主要角色基础设定 | 新角色登场、核心设定揭示 |
| `02-团队成员状态跟踪.md` | 所有团队成员当前状态、能力、物品 | 每章结束后 |
| `03-任务进度跟踪.md` | 当前任务、已完成任务、待办 | 每章结束后 |
| `04-已揭示谜团与伏笔.md` | 已揭示信息、未解谜团、已埋伏笔 | 每章结束后 |
| `05-世界观与规则文档.md` | 世界规则、地点描述、势力分布 | 新规则/地点揭示时 |

---

## 更新流程（每章必做）

### Step 1: 写作前查询
```bash
# 确认角色当前状态
grep -n "{角色名}" {truth_files.team}
# 确认任务进度
grep -n "当前任务" {truth_files.quest}
# 确认已揭示信息
grep -n "已揭示" {truth_files.mystery}
```

### Step 2: 写作中引用
- 角色能力 ≤ 已有记录
- 物品效果 ≤ 已有描述
- 地点特征 ≤ 已有设定
- 势力关系 ≤ 已有文档

### Step 3: 写作后更新（强制 — 未更新禁止开始下一章）
- 更新角色状态（受伤/升级/获得物品）
- 标记任务进展/完成
- 记录新揭示信息
- 记录新埋下的伏笔
- 如有新实体，追加到对应文件

> **阻断规则**：Truth Files 未更新前，禁止开始下一章创作。这是防止设定漂移的最后防线。

## 防漂移检查清单

### 角色一致性
- [ ] 角色当前实力与记录匹配
- [ ] 角色持有物品与记录匹配
- [ ] 角色情感状态有合理过渡
- [ ] 角色知识边界合理（不知道未揭示信息）

### 设定一致性
- [ ] 世界规则未被打破（除非是故意的规则揭示）
- [ ] 地点描述与已有记录一致
- [ ] 势力关系与已有记录一致
- [ ] 时间线无回跳/矛盾

### 任务一致性
- [ ] 当前任务与上章衔接
- [ ] 无已完成任务被重复提及为未完成
- [ ] 新任务有合理触发原因

---

## 常见漂移类型与预防

| 漂移类型 | 示例 | 预防方法 |
|----------|------|----------|
| 实力膨胀 | 主角突然展现未记录的能力 | 写作前查询能力清单 |
| 物品消失 | 上章获得的物品本章未提及 | 更新物品清单 |
| 情感跳跃 | 上章敌对本章突然信任 | 记录情感状态变化 |
| 地点矛盾 | 同一地点描述不一致 | 地点描述标准化 |
| 时间错乱 | 本章时间在上一章之前 | 每章标注时间点 |
| 知识越界 | 角色知道未揭示的信息 | 检查信息揭示记录 |

---

## 跨卷一致性特别注意事项

1. **新卷启动时**：回顾上一卷Truth Files，确认所有状态
2. **角色重新登场时**：查询其最后记录的状态
3. **旧伏笔回收时**：核对原始伏笔描述，不得篡改
4. **势力关系变化时**：追溯所有相关记录，确保一致

## 紧急修复流程

若发现已发布章节存在设定矛盾：

1. **记录矛盾**：在 Truth Files 中标注
2. **评估影响**：是否影响后续章节
3. **选择修复方式**：
   - A. 修正正文（仅当未广泛传播）
   - B. 圆场解释（在后续章节中合理化）
   - C. 接受为"未解之谜"（转为伏笔）
4. **更新 Truth Files**：记录修复决策
