# 小说章节创作流程

## 触发条件

**何时使用本技能：**
- 用户要求创作/续写小说新章节
- 用户要求修改/重写已有章节
- 用户要求检查章节质量或进行33维审计
- 用户提到"写章""续写""创作""小说"等关键词

**前置依赖：**
- 项目路径已初始化，包含 `novel-config.yaml`
- Truth Files 已创建（配置文件中指定的文件）
- 大纲文件已存在（配置文件中指定的文件）

**配置读取：**
- 技能会自动在当前目录及上级目录查找 `novel-config.yaml`
- 所有项目特定配置（路径、角色名、字数标准等）均从配置文件读取
- 不再依赖硬编码路径或角色名

---

## 字数标准

字数标准从 `novel-config.yaml` 的 `word_count` 段读取：

| 等级 | 汉字数 | 说明 |
|------|--------|------|
| **底线（阻断）** | <config.word_count.min | 禁止进入结算阶段，必须扩充 |
| **合格** | config.word_count.min ~ config.word_count.target_min-1 | 可发布但建议扩充 |
| **目标** | config.word_count.target_min ~ config.word_count.target_max | 标准目标区间 |
| **优秀** | >config.word_count.target_max | 关键章节可适当超出 |

**默认配置**：min=3000, target_min=3500, target_max=4500

## 启动新项目时

1. 使用 `references/novel-startup-template.md` 创建目录结构
2. **创建并填写 `novel-config.yaml`**（最重要！）
3. 初始化 Truth Files 模板
4. 复制 `references/config-reader.py` 到项目目录

## 项目路径

项目路径通过 `novel-config.yaml` 动态读取：

- 技能会自动查找当前目录及上级目录的 `novel-config.yaml`
- 所有文件路径从配置中读取，不再硬编码
- 支持多项目共存，切换项目只需切换目录

**示例**：
```bash
# 在万界求生目录下
cd ~/.hermes/workspace/novels/万界求生
# 运行技能 → 自动读取万界求生的配置

# 在无尽进化目录下
cd ~/writing/novels/endless-evolution
# 运行技能 → 自动读取无尽进化的配置
```

## 前置步骤

1. **读取配置**
   - 查找并读取 `novel-config.yaml`
   - 确认项目路径、文件命名规范、字数标准

2. **读取大纲**
   - 从配置 `files.outlines` 读取大纲文件
   - 定位本章情节点（数量由 `extensions.chapter_creation.plot_points_per_chapter` 决定，默认10个）

3. **读取Truth Files**（路径从 `files.truth_files` 读取）
   - `core`: 核心设定与角色
   - `team`: 团队成员状态跟踪
   - `quest`: 任务进度跟踪
   - `mystery`: 已揭示谜团与伏笔
   - `world`: 世界观与规则文档
   - 确认当前角色状态、任务进度、已揭示信息

4. **读取最近2-3章正文**
   - 从 `files.chapters_dir` 读取
   - 确认剧情衔接和叙述风格

5. **加载参考文档（分层加载）**
   - **每次必载**：`references/core-constraints.md` — 核心约束（三大定律）
   - **写作前加载**：`references/strand-weave.md` — 三线节奏控制（检查红线）
   - **写作前加载**：`references/cool-points-guide.md` — 爽点设计参考
   - **写作后审查**：`references/anti-ai-patterns.md` — AI味检测
   - **写作后审查**：`references/33-dimension-audit.md` — 33维质量审计
   - **写作后审查**：`references/review-checklist.md` — 六维审查清单
   - **项目启动时**：`references/truth-files-guide.md` — 长期一致性管理
   - **项目启动时**：`references/novel-startup-template.md` — 项目脚手架

## 核心定律（防幻觉体系）

### 三大定律（必须执行）

| 定律 | 规则 | 检查方式 |
|------|------|----------|
| **大纲即法律** | 严格执行大纲，不得擅自发挥 | 审查时对照大纲 |
| **设定即物理** | 实力/招式/物品 ≤ 已有记录 | 写作前查询Truth Files确认 |
| **发明需识别** | 新实体必须记录，不得凭空创造 | 完成后更新Truth Files |

### 章节约束分层

**Hard（必须 — 违反=重写）**
- 本章可读性达标：读者能回答"发生了什么/谁在做什么/为什么"
- 本章必须存在清晰推进：问题、目标、代价、关系变化、信息变化至少一项
- 若上章有明确承诺（钩子/未闭合问题），本章必须回应（允许部分兑现）
- 禁止输出占位正文（如 `[待补充]`、`[TODO]`、`...（省略）...`）
- 叙述人称：全书第三人称，严禁"我"叙述（对话中除外）

**Soft（建议 — 违反=记录并改进）**
- 开头尽早进入冲突/风险/强情绪（建议前200-400字）
- 未闭合问题或下一章期待锚点建议放在章末或后段
- 局面变化建议保持节奏感（800-1400字一个脉冲）
- 微兑现频率：每章至少1个"小收获"让读者感觉"这章没白看"

**Style（可选强化）**
- 对话尽量带意图（试探/回避/施压/诱导），减少纯说明句
- 避免连续大段纯解释；若必须解释，切分为"信息+行动/反应"
- 避免"回去休息了"式机械收尾；若使用平缓收尾，需保留未闭合期待

## 创作规范（强制执行）

### 结构
- 严格按10个情节点逐点创作，禁止跳过、禁止改写顺序
- 字数分配：每情节点350-450字，关键点500-600字，过渡点250-350字

### 三线节奏控制（Strand Weave）

| 线条 | 占比 | 定义 | 典型剧情 |
|------|------|------|----------|
| **Quest（主线）** | 55-65% | 核心任务、升级、战斗、夺宝 | 规则破解、世界探索、冲突对抗 |
| **Fire（感情线）** | 20-30% | 情感关系发展（爱情/友情/师徒） | 团队互动、信任建立、情感转折 |
| **Constellation（世界观线）** | 10-20% | 扩展设定、展示新势力/地点 | 规则揭示、新势力登场、背景展开 |

**交织红线：**
- Quest 连续不超过 5 章
- Fire 断档不超过 10 章
- Constellation 断档不超过 15 章

### Anti-AI 写作规范（去AI味）

**词汇层：**
- 禁止高频AI词汇密集："缓缓/淡淡/微微"+动词在500字内出现≤1次
- 禁止"眸中闪过""瞳孔微缩"等神态模板密集使用
- 删掉万能副词（缓缓/淡淡/微微），用具体动作替代

**句式层：**
- 禁止"起因→经过→结果→感悟"四段闭环
- 禁止连续同构句（≥3句主谓宾结构一致）
- 禁止每段都以总结句收尾（"他终于明白了""由此可见"）
- 禁止同一信息用不同句式重复说2-3遍

**叙事层：**
- 禁止节奏匀速（段落信息密度过于均匀，无快慢之分）
- 禁止"他不知道的是……""殊不知……"戏剧性反讽提示
- 章末禁止"安全着陆"（冲突完美解决，无遗留不安感）
- 禁止展示后紧跟解释（先用动作展示，紧接着一句话解释含义）

**情感层：**
- 情绪描写禁止标签化（"他感到愤怒""她非常紧张"）
- 必须用行为/生理暗示替代（手虚握符箓、进门前神识扫一圈）
- 禁止情绪即时切换（上句愤怒，下句就平静，无过渡）
- 禁止所有角色用同一套反应模板

**对话层：**
- 禁止对话为信息宣讲（解释背景而非推进冲突）
- 禁止全员书面语、无口语特征、无个人口癖
- 禁止对白后跟解释性叙述（"他这么说是因为……"）
- 对话必须带潜台词和意图冲突，有抢话、沉默、答非所问

### 句式模板化禁忌（致命问题）

**绝对禁止的句式模板：**

1. **"是……"停顿句式** — 最严重
   - 错误：`"世界，是……故事。"` `"生命，是……角色。"`
   - 正确：`"世界是故事的集合。"` `"生命是故事中的角色。"`
   - 规则：禁止在"是"字前后用省略号制造刻意停顿

2. **"不是……是……"对比句式** — 高频泛滥
   - 错误：`"不是规则，是……故事。"`
   - 正确：`"世界不是规则的集合，而是故事的集合。"`
   - 规则：同一章内不得超过3处，且必须分散

3. **"像……像……像……"排比比喻** — 空洞堆砌
   - 错误：`"像种子，像花朵，像某种……存在。"`
   - 正确：用一个精准比喻，然后展开描写

4. **"某种……"模糊指代** — 逃避具体
   - 错误：`"某种……更本质的东西。"`
   - 正确：直接写出具体是什么

5. **"不是……不是……不是……"否定排比**
   - 错误：`"不是生命，不是机器，不是任何已知的形态。"`
   - 正确：`"它不是生命，也不是机器，而是一种从未被记录的机制。"`

6. **"从……到……从……到……"扩散句式**
   - 错误：`"从手指，到手臂。从手臂，到身体。"`
   - 正确：`"变化从手指开始，蔓延到手臂，再到身体。"`

**句式多样性要求：**
- 同一章内，任何句式模板不得重复超过2次
- 连续3段不得使用相同的段落结构
- 对话、叙述、描写的句式必须有明显差异

### 对话要求（去AI感专项）

**核心原则：对话不是信息传递，是人物关系的战场。**

**绝对禁止：**
- 极简问答式对话（"名字？""{主角名}。""目的？""求生。"）
- 单句来回超过2轮
- 一问一答式信息灌输
- 角色干站着对话，无动作穿插

**技巧库（根据场景合理穿插）：**
1. **动作穿插**：说话时同时在做事、有反应、有停顿
2. **潜台词**：让角色说A意思是B，不直抒胸臆
3. **打断与省略**：真实对话有半截话、突然沉默、被打断
4. **观点摩擦**：角色互相不服、互相试探
5. **环境插入**：用第三者、环境变化打断封闭对话
6. **信息延迟**：重要信息慢慢漏出，不是一问就答
7. **角色差异**：每个角色有独特语气、用词、口头禅

**口吻差异化标准：**

| 角色类型 | 句长 | 词汇标记 | 示例 |
|---------|------|---------|------|
| 暴躁少年 | 短句，断句多 | 口语化，带语气词 | "滚！别废话！" |
| 老学究 | 长句，从句多 | 文言痕迹，引经据典 | "此事……若依老夫所见，恐怕不妥。" |
| 市井商人 | 中等，话术感 | 客套+算计 | "张爷您说笑了，小本生意，哪敢怠慢。" |
| 冷静领袖 | 中等，克制 | 理性分析，少情绪词 | "先确认损失，再决定下一步。" |

### 动作要求
- 必须分解步骤：困难、尝试、失败、调整
- 禁止一步完成

### 场景要求
- 必须有层次：次要角色反应、环境变化、旁观者视角

### 感官要求
- 必须具体化：视觉/听觉/触觉细节
- 禁止抽象概括

## 爽点与节奏工程

### 六种爽点执行模式

| 模式 | 结构 | 适用 |
|------|------|------|
| 装逼打脸 | 对方轻视→主角展示实力→对方震惊/后悔 | 都市、玄幻 |
| 扮猪吃虎 | 表面弱小→关键时刻爆发→众人惊艳 | 重生、隐藏身份 |
| 越级反杀 | 实力差距明显→主角逆袭→敌人不可置信 | 玄幻、竞技 |
| 打脸权威 | 权威质疑→主角用实力证明→权威认可/尴尬 | 职场、学院 |
| 反派翻车 | 反派得意→计划破产→反派狼狈 | 所有题材 |
| 甜蜜超预期 | 平淡日常→意外惊喜→情感升温 | 言情、甜宠 |

### 爽点三段式结构（30/40/30参考框架）

**铺垫阶段（30%）：** 建立预期、制造反差、设置信息差
**兑现阶段（40%）：** 触发时机、展现方式、情绪高峰
**余波阶段（30%）：** 反应、收获、新期待

### 微兑现（Micro-Payoff）

每章必须给读者"小收获"：

| 类型 | 描述 | 示例 |
|------|------|------|
| 信息兑现 | 揭示新信息/线索 | "原来那把钥匙的真正用途是..." |
| 关系兑现 | 关系推进/确认 | "她第一次主动握住了他的手" |
| 能力兑现 | 能力提升/新技能 | "他终于掌握了这门功法的精髓" |
| 情绪兑现 | 情绪释放/共鸣 | "他终于说出了压在心底的那句话" |
| 线索兑现 | 伏笔回收/推进 | "三年前的那件事，终于有了眉目" |

### 钩子类型（章末驱动）

| 类型 | 定义 | 示例 |
|------|------|------|
| 危机钩 | 敌人出现/危险逼近 | "门后传来女儿的哭声，但规则写着：不要开门" |
| 悬念钩 | 信息缺口/未解之谜 | "监控里那个身影，分明是三天前已经死去的人" |
| 渴望钩 | 好事即将发生/奖励可期 | "明日宗门大比，所有嘲笑过他的人都将见证他的崛起" |
| 情绪钩 | 触发强烈情绪反应 | "她终于明白，那个从不解释的男人一直在保护她" |
| 选择钩 | 两难抉择/高风险决策 | "吞下这颗丹药，有三成概率突破，七成概率走火入魔" |

## 正确扩充方式
- 对话交锋扩展：性格冲突、意见分歧、说服过程
- 动作细节分解：具体步骤、技术细节、时间压力
- 场景层次增加：环境变化、旁观者反应、对比冲击
- 副线剧情推进：配角小冲突、新信息揭示
- 感官描写具体化：精确数字、具体动作、对比记忆
- 信息增量扩充：新线索、新发现、新转折

## 六维审查清单（创作后自检 — 基础层）

> 六维审查是33维审计的简化版，用于快速自检。完整审计请使用 `references/33-dimension-audit.md`。
> 
> **维度映射关系：**
> - 六维"设定一致性" → 33维"基础质量"+部分"叙事技巧"
> - 六维"时间线" → 33维"基础质量"（时间衔接）
> - 六维"叙事连贯" → 33维"结构节奏"（钩子回应）+"基础质量"（推进感）
> - 六维"角色一致性" → 33维"叙事技巧"（对话意图、角色反应）+"对话质量"（角色差异）
> - 六维"逻辑" → 33维"基础质量"（可读性、推进感）
> - 六维"AI味检测" → 33维"语言风格"+"句式模板"+"对话质量"
> 
> **使用建议：**
> - 日常快速检查 → 六维审查（review-checklist.md）
> - 发布前完整审计 → 33维审计（33-dimension-audit.md）
> - 质量争议/评分 → 33维审计（有评分等级A/B/C/D）

### 1. 设定一致性（Consistency）
- [ ] 角色能力是否与当前状态匹配
- [ ] 地点描述是否与世界观一致
- [ ] 物品/规则使用是否符合已建立设定

### 2. 时间线（Timeline）
- [ ] 本章时间是否与上章衔接（无回跳）
- [ ] 倒计时/截止日期是否正确推进
- [ ] 无角色同时出现在两个地点

### 3. 叙事连贯（Continuity）
- [ ] 上章钩子是否有回应
- [ ] 场景转换是否有过渡
- [ ] 情绪弧是否连续

### 4. 角色一致性（Character）
- [ ] 对话风格是否符合角色特征
- [ ] 行为是否与已建立的性格/动机一致
- [ ] 角色知识边界合理（不使用不应知道的信息）

### 5. 逻辑（Logic）
- [ ] 因果关系是否成立
- [ ] 角色决策是否有合理动机
- [ ] 冲突结果是否符合力量对比

### 6. AI味检测（AI Flavor）
- [ ] 词汇层：无高频AI词汇密集（缓缓/淡淡/微微 500字内≤1次）
- [ ] 句式层：无"起因→经过→结果→感悟"四段闭环
- [ ] 叙事层：节奏有快慢变化，无章末安全着陆
- [ ] 情感层：情绪用行为暗示，无标签化（他感到X ≤2处/章）
- [ ] 对话层：对话带意图冲突，无信息宣讲

## 自检流程

### 快速检测命令（一键运行）
```bash
echo "=== 基础质量 ==="
echo "字数:"
grep -oP '[\x{4e00}-\x{9fff}]' 第XXX章-标题.md | wc -l
echo "占位符:"
grep -cP '\[待补充\]|\[TODO\]|\.\.\.（省略）\.\.\.' 第XXX章-标题.md

echo "=== 语言风格（阈值见33维审计） ==="
echo "万能副词:"
grep -oE '缓缓|淡淡|微微|轻轻' 第XXX章-标题.md | wc -l
echo "神态模板:"
```
